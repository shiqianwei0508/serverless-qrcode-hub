## 协议选择弹窗 + 复制兜底修复
- **协议选择弹窗**：用户要求保存时若地址无协议，弹窗让用户选 https 还是 http（选否则补 http）。实现：
  - `dist/admin.html` 新增 `#protocol-modal` 弹窗（标题/提示 + "补全为 https://"、"补全为 http://"、Cancel 三按钮）。
  - 新增 `askProtocolPrefix(rawTarget)` 返回 Promise，解析为补全后的 URL 或 null（取消）。在 `addMapping` 与 `saveEditFromModal` 保存前检测 `!/^https?:\/\//i.test(target)/`，无协议则 await 弹窗；取消则 return 不保存；选则 `finalTarget` 入参。
  - i18n.js 8 语种新增 `app.targetNoScheme`/`app.targetNoSchemeHint`/`app.prefixHttps`/`app.prefixHttp`。
  - 后端 `normalizeTarget()`（补 https）保留为安全兜底（经前端弹窗后 target 已带协议，故后端 targetNormalized 通常为 false，旧的 targetAutoHttps 提示基本不再触发）。
- **复制"提示已复制但剪贴板为空"修复**：根因不是安全上下文，而是复制按钮位于 `detail-modal`(`<dialog>` 用 `showModal()` 打开)内部。模态 dialog 打开时文档其余部分变 inert，之前兜底把隐藏 textarea 追加到 `document.body`(inert 区)导致 `execCommand('copy')` 静默失败(部分浏览器仍返回 true)，于是误报"已复制"。修复：兜底 textarea 追加到当前打开的 `dialog[open]`(`document.querySelector('dialog[open]') || document.body`)，并加 `setSelectionRange(0, length)` 确保全选。手机端因走 `navigator.clipboard`(安全上下文)不受影响。
- **Toast 被详情弹窗挡住修复**：`<dialog showModal()>` 处于 top layer，普通 `z-index` 压不过，且 `document.body` 内元素(含 `#alertContainer`)会渲染在 dialog 之后。修复：`showAlert()` 在弹窗打开时把 `#alertContainer` 移入当前 `dialog[open]`(top-layer 内)；`position:fixed` 仍锚定视口(因 dialog 非 fixed 包含块)。这样"链接已复制"等提示浮于弹窗之上。

## 添加条目输入校验加固
- 用户问"添加条目是否验证了"。原状：前端 `addMapping` 有校验(name/path/target 必填、path 仅 `[a-zA-Z0-9-_]+`、target 协议弹窗、微信需图)；但**后端 `createMapping`/`updateMapping` 只检查非空 + 保留词 + 过期数字 + 微信图**，没有 path 格式/长度/目标协议校验——绕过前端直接调 API 可写任意路径/超长内容。
- 加固(`index.js`)：新增 `validateMappingInput(path, target, name, expiry)`（在 `createMapping`、`updateMapping` 中 `banPath` 检查之后调用）。规则：path 必须匹配 `/^[a-zA-Z0-9-_]+$/`(否则 `PATH_INVALID`)；path≤64 / name≤128 / target≤2048(否则 `INPUT_TOO_LONG`)；target 必须 `new URL()` 可解析且协议为 http/https(否则 `TARGET_INVALID`，挡住 javascript:/data: 等)。常量 `PATH_MAX/NAME_MAX/TARGET_MAX/PATH_RE`。
- `i18n.js` 8 语种新增 `app.targetInvalid`/`app.inputTooLong`；`admin.html` 的 `ERROR_CODE_KEYS` 新增 `PATH_INVALID→app.pathInvalid`、`TARGET_INVALID→app.targetInvalid`、`INPUT_TOO_LONG→app.inputTooLong` 映射。
- XSS：后端 expired/wechat 页用 `escapeHtml(name)`，前端卡片用 `escapeHtml(nameDisplay)`、详情弹窗用 `textContent`，均已安全。
- **复制按钮 PC 不生效修复**：原 `navigator.clipboard.writeText` 仅安全上下文（HTTPS/localhost）可用，局域网 http 访问（PC）会失败；手机端正常。新增 `copyToClipboard()`：优先 clipboard API（且 `window.isSecureContext`），失败/不可用时回退 `fallbackCopyText()`（隐藏 textarea + `document.execCommand('copy')`）。`copyTextToClipboard` 与 `copyDecodedText` 均改用之。
